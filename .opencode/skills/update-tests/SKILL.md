## Purpose

This skill keeps testing infrastructure and documentation synchronized with codebase changes. Ensures LLMs have accurate information about testing workflows, commands, and verification patterns.

## When to Use

Apply this skill when:

- **Adding new features** that require tests or test setup
- **Changing test commands** in `package.json`
- **Updating testing libraries** or frameworks
- **Modifying LLM testing workflows** (Chrome DevTools, visual verification)
- **Before major releases** to verify test coverage documentation
- **After adding mock data scripts** or test utilities

## Workflow

### Phase 1: Analyze Test Infrastructure

**1. Read package.json test scripts:**

```bash
# Extract all test-related scripts
- test:user:setup
- test:spoof:twitch
- test:spoof:youtube
- test:spoof:kick
- test:spoof:managed-basic
- test:visual:setup
- Any other test:* commands
```

**2. Check test setup scripts:**

```bash
# List scripts directory
scripts/
├── setup-test-user.js       # Test user creation
├── spoof-platform-data.js   # Mock platform data
├── spoof-youtube-connected.js # YouTube mock data
└── spoof-managed-basic.js   # Managed basic state mock
```

**3. Review test utilities:**

- Check `src/lib/test/` (if exists)
- Review test helpers in `src/lib/`
- Check for Vitest/Jest config files
- Review Chrome DevTools testing patterns

**4. Check environment setup:**

- Read `.env.example` for test credentials
- Verify `TEST_USER_EMAIL`, `TEST_USER_PASSWORD`, `TEST_USER_ID`
- Check test database configuration

### Phase 2: Review LLM Testing Workflows

**1. Chrome DevTools Testing:**

- Navigation workflows (`chrome-devtools_navigate_page`)
- Screenshot verification (`chrome-devtools_take_screenshot`)
- Scripting-based validation (`chrome-devtools_evaluate_script`)
- Responsive testing (`chrome-devtools_resize_page`)

**2. Visual Verification Patterns:**

- Vision-capable model workflows
- Non-vision model scripting validation
- Viewport testing (mobile, tablet, desktop)
- Screenshot asset management (`.repo/temp/` vs `.repo/screenshots/`)

**3. Test Data Management:**

- Mock data generation scripts
- Test user setup process
- Platform spoofing workflows
- Test credentials storage

**4. Current Testing Documentation:**

- README.md testing section
- AGENTS.md testing workflows
- Skill documentation for testing-related skills

````
- Update "For Vision-Capable Models" section

- Explain test user setup process

**3. Update Skills (if needed):**

If testing workflows changed significantly:
- Update `update-docs` skill testing examples
- Update `deep-update-docs` skill verification phase

Add or update test-related variables:

````

### Phase 4: Verify Test Commands

**1. Test Command Availability:**

```bash
# Verify all documented commands exist
pnpm test:user:setup --if-present
pnpm test:spoof:twitch --if-present
pnpm test:spoof:youtube --if-present
pnpm test:visual:setup --if-present
```

**2. Check Script Dependencies:**

- Verify Node.js scripts have correct imports
- Check Supabase CLI is available
- Verify test user permissions

**3. Validate Documentation:**

- All commands in docs exist in `package.json`
- All script paths are correct
- Test credentials documented accurately
- Workflows match current codebase

## Guidelines

- **ACCURATE**: Verify all commands work before documenting
- **COMPLETE**: Include all test scripts, not just some
- **CURRENT**: Match workflows to actual codebase state
- **LLM-FRIENDLY**: Write for AI agents who will read docs
- **MAINTAINABLE**: Keep test docs easy to update
